Markdown 是一种轻量级标记语言，语法简洁直观，广泛用于文档编写、笔记、博客等场景。以下是 **完整且常用的 Markdown 语法格式**，按功能分类整理，包含示例和注意事项，方便快速查阅和使用。


### 一、标题（Headings）
通过 `#` 符号定义，`#` 数量对应标题级别（1-6级），`#` 后需加**空格**，否则不生效。

| 语法                  | 效果（预览）               | 说明                     |
|-----------------------|----------------------------|--------------------------|
| `# 一级标题`          | # 一级标题                 | 最高级，字体最大         |
| `## 二级标题`         | ## 二级标题                | 常用作文章大章节标题     |
| `### 三级标题`        | ### 三级标题               | 常用作小节标题           |
| `#### 四级标题`       | #### 四级标题              | 字体较小，用于细分内容   |
| `##### 五级标题`      | ##### 五级标题             | -                        |
| `###### 六级标题`     | ###### 六级标题            | 最低级，字体接近正文     |

**注意**：一级标题建议整篇文档只使用1个，保证结构清晰。


### 二、文本样式（Text Styling）
用于强调、标记文本，支持粗体、斜体、删除线等。

| 语法                  | 效果（预览）               | 说明                     |
|-----------------------|----------------------------|--------------------------|
| `*斜体文本*` 或 `_斜体文本_` | *斜体文本* / _斜体文本_    | 两种符号均可，推荐用 `*` |
| `**粗体文本**` 或 `__粗体文本__` | **粗体文本** / __粗体文本__ | 强调重点内容             |
| `***粗斜体文本***` 或 `___粗斜体文本___` | ***粗斜体文本*** | 同时加粗和斜体           |
| `~~删除线文本~~`      | ~~删除线文本~~             | 标记已废弃或修正的内容   |
| `==高亮文本==`        | ==高亮文本==               | 部分平台支持（如 Typora、语雀） |
| `H~2~O`               | H~2~O                      | 下标（如化学公式、日期） |
| `X^2^`                | X^2^                       | 上标（如数学公式、幂次） |


### 三、段落与换行（Paragraphs & Line Breaks）
Markdown 对段落和换行的处理有明确规则，避免格式混乱。

1. **段落**：  
   直接输入文本即可形成段落，段落之间需留**一个空行**（即按两次 `Enter`），否则会被视为同一段落。  
   - 示例：  
     ```markdown
     这是第一个段落。内容可以换行输入，但只要不手动留空行，最终会合并为一段。
     （这里即使手动换行了，预览时仍会显示为连续文本）

     这是第二个段落。（与上一段之间留了空行，会被识别为独立段落）
     ```

2. **强制换行**：  
   若需在同一段落内换行（不拆分段落），可在换行处加 **2个及以上空格 + Enter**，或使用 `<br>` 标签。  
   - 示例：  
     ```markdown
     第一行末尾加两个空格  
     这行会换行显示（同一段落内）

     第一行末尾用<br>
     这行也会换行显示
     ```


### 四、列表（Lists）
支持有序列表（带数字）和无序列表（带符号），可嵌套使用。

#### 1. 无序列表（Unordered Lists）
用 `*`、`+`、`-` 作为列表标记，标记后需加**空格**，三种符号效果一致，推荐统一用 `*`。  
- 示例：  
  ```markdown
  * 列表项1
  * 列表项2
    * 嵌套列表项2.1（前面加 4个空格 或 1个Tab）
    * 嵌套列表项2.2
  * 列表项3
  ```
- 预览效果：  
  * 列表项1
  * 列表项2
    * 嵌套列表项2.1
    * 嵌套列表项2.2
  * 列表项3

#### 2. 有序列表（Ordered Lists）
用 `数字 + .` 作为标记，标记后需加**空格**，数字无需手动排序（Markdown 会自动修正）。  
- 示例：  
  ```markdown
  1. 第一步
  2. 第二步
     1. 嵌套步骤2.1（前面加 4个空格）
     2. 嵌套步骤2.2
  3. 第三步
  ```
- 预览效果：  
  1. 第一步
  2. 第二步
     1. 嵌套步骤2.1
     2. 嵌套步骤2.2
  3. 第三步

#### 3. 任务列表（Task Lists）
部分平台（如 GitHub、Typora）支持，用 `- [ ]` 表示未完成，`- [x]` 表示已完成，括号内需加**空格**。  
- 示例：  
  ```markdown
  - [x] 完成Markdown语法学习
  - [ ] 编写示例文档
  - [ ] 检查格式正确性
  ```
- 预览效果：  
  - [x] 完成Markdown语法学习
  - [ ] 编写示例文档
  - [ ] 检查格式正确性


### 五、引用（Blockquotes）
用于引用他人观点、文档内容或强调某段文字，用 `>` 作为标记，支持嵌套和多行引用。

1. **基础引用**：  
   `>` 后加空格，引用内容可换行，换行时无需重复加 `>`（但部分平台需加）。  
   - 示例：  
     ```markdown
     > 这是一段引用文本，用于标记外部内容或重要说明。
     > 换行时继续加 > 可确保格式统一（推荐）。
     ```
   - 预览效果：  
     > 这是一段引用文本，用于标记外部内容或重要说明。
     > 换行时继续加 > 可确保格式统一（推荐）。

2. **嵌套引用**：  
   用多个 `>` 实现嵌套（如 `>>` 表示二级引用）。  
   - 示例：  
     ```markdown
     > 一级引用
     >> 二级引用（嵌套在一级内）
     >>> 三级引用（嵌套在二级内）
     ```
   - 预览效果：  
     > 一级引用
     >> 二级引用（嵌套在一级内）
     >>> 三级引用（嵌套在二级内）

3. **引用内混排**：  
   引用内可嵌套列表、代码块等元素。  
   - 示例：  
     ```markdown
     > 引用内的列表：
     > * 列表项1
     > * 列表项2
     > 
     > 引用内的代码：`const a = 1`
     ```
   - 预览效果：  
     > 引用内的列表：
     > * 列表项1
     > * 列表项2
     > 
     > 引用内的代码：`const a = 1`


### 六、代码（Code）
用于展示代码片段，支持**行内代码**和**代码块**，部分平台支持语法高亮。

#### 1. 行内代码（Inline Code）
用 **1个反引号**（`）包裹代码，反引号位于键盘 `Tab` 键上方（英文输入法）。  
- 示例：  
  ```markdown
  定义变量 `const name = "Markdown"`，然后调用 `printName()` 函数。
  ```
- 预览效果：  
  定义变量 `const name = "Markdown"`，然后调用 `printName()` 函数。

#### 2. 代码块（Code Blocks）
有两种方式，推荐用 **三个反引号**（```），支持指定语言实现语法高亮。

- 方式1：三个反引号（推荐）  
  开头用 ``` + 语言名称（如 javascript、python），结尾用 ```，语言名称不区分大小写。  
  - 示例（JavaScript 代码）：  
    ```markdown
    ```javascript
    // 计算两数之和
    function add(a, b) {
      return a + b;
    }
    const result = add(2, 3);
    console.log(result); // 输出 5
    ```
    ```
  - 预览效果（带语法高亮）：  
    ```javascript
    // 计算两数之和
    function add(a, b) {
      return a + b;
    }
    const result = add(2, 3);
    console.log(result); // 输出 5
    ```

- 方式2：缩进（不推荐）  
  代码块前加 **4个空格** 或 **1个Tab**，但不支持语法高亮，仅适合简单代码。  
  - 示例：  
    ```markdown
        # Python 简单代码
        print("Hello, Markdown!")
    ```
  - 预览效果：  
        # Python 简单代码
        print("Hello, Markdown!")


### 七、链接（Links）
支持**行内链接**和**参考链接**，用于跳转至外部页面或内部锚点。

#### 1. 行内链接（Inline Links）
格式：`[链接文本](链接地址 "可选的标题")`，标题在鼠标 hover 时显示。  
- 示例：  
  ```markdown
  访问 [GitHub](https://github.com "GitHub 官网") 查看开源项目，或查看 [Markdown 官方文档](https://daringfireball.net/projects/markdown/)。
  ```
- 预览效果：  
  访问 [GitHub](https://github.com "GitHub 官网") 查看开源项目，或查看 [Markdown 官方文档](https://daringfireball.net/projects/markdown/)。

#### 2. 参考链接（Reference Links）
适合链接多次复用的场景，先定义链接标识，再在文档任意位置引用。  
- 格式：  
  1. 引用部分：`[链接文本][链接标识]`  
  2. 定义部分：`[链接标识]: 链接地址 "可选标题"`（定义可放在文档末尾，不影响预览）  
- 示例：  
  ```markdown
  推荐学习 [Markdown][md] 和 [VS Code][vsc]，[VS Code][vsc] 对 Markdown 支持很好。

  // 链接定义（可放在文档末尾）
  [md]: https://daringfireball.net/projects/markdown/ "Markdown 官方文档"
  [vsc]: https://code.visualstudio.com/ "VS Code 官网"
  ```
- 预览效果：  
  推荐学习 [Markdown][md] 和 [VS Code][vsc]，[VS Code][vsc] 对 Markdown 支持很好。

#### 3. 锚点链接（Anchor Links）
用于跳转到文档内的指定标题（仅部分平台支持，如 GitHub、Typora）。  
- 规则：锚点地址为 **标题的小写形式，空格替换为 `-`**（若标题有特殊符号，需去除或转义）。  
- 示例：  
  ```markdown
  点击跳转至 [「列表」部分](#四、列表lists)
  ```
- 预览效果：点击跳转至 [「列表」部分](#四、列表lists)（实际使用时需确保标题格式匹配）。


### 八、图片（Images）
语法与链接类似，仅多一个 `!` 前缀，格式：`![图片加载失败时的提示文本](图片地址 "可选标题")`。

#### 1. 网络图片（推荐）
直接使用图片的在线 URL（需确保 URL 可访问）。  
- 示例：  
  ```markdown
  ![VS Code 图标](https://code.visualstudio.com/favicon.ico "VS Code 图标")
  ```
- 预览效果：  
  ![VS Code 图标](https://code.visualstudio.com/favicon.ico "VS Code 图标")

#### 2. 本地图片
使用本地文件路径（相对路径或绝对路径），仅在本地编辑器（如 Typora）中生效，上传到平台需先上传图片。  
- 示例（相对路径，图片与文档在同一文件夹）：  
  ```markdown
  ![我的截图](./screenshot.png "本地截图")
  ```


### 九、表格（Tables）
语法需注意对齐方式，表头与内容之间需用 `| --- |` 分隔，支持左对齐、右对齐、居中对齐。

#### 基础格式
| 语法                  | 预览效果                   |
|-----------------------|----------------------------|
| ```markdown
  | 姓名 | 年龄 | 职业 |
  | --- | --- | --- |
  | 张三 | 25 | 工程师 |
  | 李四 | 30 | 设计师 |
  ``` | | 姓名 | 年龄 | 职业 |
| --- | --- | --- |
| 张三 | 25 | 工程师 |
| 李四 | 30 | 设计师 | |

#### 对齐方式
在 `---` 前后加 `:` 控制对齐：
- `:---` 或 `---:`：左对齐 / 右对齐
- `:---:`：居中对齐
- 不加分号：默认左对齐

- 示例（带对齐）：  
  ```markdown
  | 商品 | 价格（元） | 库存 |
  | :--- | ---: | :---: |
  | 手机 | 3999 | 50 |
  | 电脑 | 5999 | 30 |
  ```
- 预览效果：  
  | 商品 | 价格（元） | 库存 |
  | :--- | ---: | :---: |
  | 手机 | 3999 | 50 |
  | 电脑 | 5999 | 30 |

**注意**：`|` 两侧的空格可省略，表格内支持文本样式（如粗体 `**`）。


### 十、分割线（Horizontal Rules）
用 `***`、`---` 或 `___` 表示，需单独占一行，前后建议留空行，三种符号效果一致。  
- 示例：  
  ```markdown
  上文内容

  ***

  下文内容
  ```
- 预览效果：  
  上文内容

  ***

  下文内容


### 十一、特殊符号（Escaping Characters）
若需显示 Markdown 语法符号（如 `#`、`*`、`|`），需在符号前加 **反斜杠（\）** 转义。  
- 示例：  
  ```markdown
  显示 # 符号：\#  
  显示 * 符号：\*  
  显示 | 符号：\|  
  ```
- 预览效果：  
  显示 # 符号：\#  
  显示 * 符号：\*  
  显示 | 符号：\|  


### 十二、扩展语法（部分平台支持）
以下语法并非所有 Markdown 编辑器都支持（需依赖插件或特定平台，如 Typora、GitHub、语雀）：

1. **脚注（Footnotes）**  
   格式：`文本[^脚注标识]` + 末尾定义 `[^脚注标识]: 脚注内容`  
   - 示例：  
     ```markdown
     Markdown 由 John Gruber 创造[^1]。

     [^1]: John Gruber 是美国程序员、作家，2004年与 Aaron Swartz 合作设计了 Markdown。
     ```
   - 预览效果：  
     Markdown 由 John Gruber 创造[^1]。

2. **数学公式（Math Equations）**  
   用 `$` 包裹行内公式，`$$` 包裹块级公式（需编辑器支持 LaTeX 语法）。  
   - 示例：  
     ```markdown
     行内公式：$E=mc^2$  
     块级公式：
     $$
     \sum_{i=1}^n i = \frac{n(n+1)}{2}
     $$
     ```
   - 预览效果：  
     行内公式：$E=mc^2$  
     块级公式：
     $$
     \sum_{i=1}^n i = \frac{n(n+1)}{2}
     $$


### 常用编辑器推荐
- 本地编辑器：Typora（所见即所得，支持所有扩展语法）、VS Code（需装 Markdown 插件）  
- 在线平台：GitHub（支持标准语法+扩展）、语雀、Notion（语法略有差异，需适配）

掌握以上语法，即可满足 99% 的 Markdown 文档编写需求，建议结合实际编辑器多练习，熟悉格式细节～

[^1]: John Gruber 是美国程序员、作家，2004年与 Aaron Swartz 合作设计了 Markdown。
