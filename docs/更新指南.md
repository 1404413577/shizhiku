非常简单！因为我们已经配置好了 GitHub Actions 自动化工作流，以后您**完全不需要手动部署**了。

当您在本地修改了代码（比如添加了新文档、修改了样式、修复了 Bug 等）之后，只需要按照正常的 Git 流程把代码推送到 GitHub 即可：

```bash
# 1. 添加所有更改
git add .

# 2. 提交更改，写上您的修改说明
git commit -m "feat: 更新了一些文档内容"

# 3. 推送到 GitHub 的 main 分支
git push github main
```

**发生了什么？**
只要您一执行 `git push github main` 把代码推送到远程的 `main` 分支，GitHub Actions 就会自动被触发。
它会在后台帮您：
1. 下载最新的代码
2. 运行 `npm install` 安装依赖
3. 运行 `npm run build` 打包项目
4. 自动把打包好的文件推送到 `gh-pages` 分支并更新网站

**您只需要：**
推送代码后，稍等 1-2 分钟，刷新一下您的网页 `https://1404413577.github.io/shizhiku/`，就能看到最新的内容了！
