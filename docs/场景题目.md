# 场景题目

### 1.请求失败会弹出一个toast,如何在批量请求失败，只弹出一个toast
首先 我的思路是
- 实现 “批量请求失败只弹出一个 alert” 的核心思路可以概括为：“统一监控状态，延迟统一提示”，具体分为以下几个步骤：
标记失败状态
定义一个标志位（如 hasError 布尔值）或计数器（如 errorCount 数字），用于记录批量请求中是否有失败情况。
捕获单个请求的失败
对每个请求的失败情况进行单独捕获（使用 .catch()），在捕获到失败时，更新标志位或计数器（例如将 hasError 设为 true，或 errorCount 加 1），但不立即弹出提示。
等待所有请求完成
使用 Promise.all() 等待批量请求中的所有请求（无论成功或失败） 都执行完毕。这里需要注意：每个请求的失败要被捕获（避免 Promise.all() 直接进入 reject 状态），确保能收集到所有请求的结果。
统一判断并提示
所有请求完成后，根据标志位或计数器的状态，只执行一次提示：
若有失败（hasError 为 true 或 errorCount > 0），弹出错误提示；
若全部成功，则弹出成功提示（可选）。
这种思路的关键在于：不针对单个请求的失败立即提示，而是等所有请求结束后 “汇总判断”，从而避免多次弹出提示，同时保证批量请求能完整执行。

```javascript
<template>
  <div class="batch-request-example">
    <button @click="fetchBatchData" class="fetch-btn">
      批量获取数据
    </button>
  </div>
</template>

<script>
import { ref } from 'vue';

export default {
  setup() {
    // 模拟API请求
    const fetchData = async (id) => {
      return new Promise((resolve, reject) => {
        // 随机模拟成功或失败
        setTimeout(() => {
          if (Math.random() > 0.5) {
            resolve({ id, data: `数据${id}` });
          } else {
            reject(new Error(`请求${id}失败`));
          }
        }, 500);
      });
    };
    
    // 批量请求处理
    const fetchBatchData = async () => {
      const requestIds = [1, 2, 3, 4, 5]; // 需要请求的ID列表
      let hasError = false; // 错误标志位
      
      // 创建所有请求的Promise
      const requests = requestIds.map(id => 
        fetchData(id)
          .catch(error => {
            console.error(error.message);
            hasError = true; // 标记有错误发生
          })
      );
      
      // 等待所有请求完成
      await Promise.all(requests);
      
      // 如果有错误，只显示一次alert
      if (hasError) {
        alert('批量请求完成，部分请求失败');
      } else {
        alert('所有请求成功完成');
      }
    };
    
    return {
      fetchBatchData
    };
  }
};
</script>
```
