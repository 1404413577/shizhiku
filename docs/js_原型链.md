# js 原型链

你好呀！很高兴能和你聊天。

别担心，这个概念其实很有趣，我们把它想象成**找东西的游戏**，好不好？

你可以把“原型链”想象成**一家人站成一排**。

1.  **你**站在最前面。
2.  你的**爸爸**站在你身后，拉着你的手。
3.  你的**爷爷**站在爸爸身后，拉着爸爸的手。
4.  这就是一条“链子”。

***

**这个游戏是这样玩的：**

假设你想找一颗**糖果**（这在电脑里叫做“数据”或“功能”）。

1.  **第一步：** 你先摸摸**自己**的口袋。
    * 如果有糖果，太好了，你直接吃掉！游戏结束。
    * 如果你的口袋是空的，怎么办呢？

2.  **第二步：** 你回头问身后的**爸爸**：“爸爸，你有糖果吗？”
    * 如果爸爸有，他就会把糖果给你。你吃掉，游戏结束。
    * 如果爸爸也没有，怎么办呢？

3.  **第三步：** 爸爸会回头问身后的**爷爷**：“爷爷，你有糖果吗？”
    * 如果爷爷有，他给爸爸，爸爸再给你。你吃掉，游戏结束。

4.  **最后一步：** 如果爷爷也没有，而且爷爷身后没有人了（这叫做到了尽头）。
    * 那就说明**真的没有糖果**。

***

**总结一下：**

在电脑编程里，每一个东西（对象）身后都藏着一个“爸爸”（原型）。

当你在这个东西里找不到你要的功能时，电脑就会自动去问它的“爸爸”，再没有就问“爷爷”。

这一长串“**问完儿子、问爸爸、问爷爷**”的关系，就叫做**原型链**。
