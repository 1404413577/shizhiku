# 静态文档访问功能使用说明

## 功能概述

知识库应用现在支持静态文档访问功能，可以将 `docs/` 文件夹中的 Markdown 文档自动加载到应用中作为预设文档。

## 实现原理

### 构建时处理
- 使用 Vite 插件在构建时扫描 `docs/` 文件夹
- 将所有 `.md` 文件转换为 JSON 格式
- 生成 `preset-docs.json` 文件包含在构建产物中

### 开发时处理
- 使用虚拟模块机制在开发时提供文档内容
- 支持热更新和实时预览

### 应用启动时加载
- 检查是否已加载过预设文档
- 如果没有文档或缺少预设文档，自动加载
- 将预设文档保存到本地存储中

## 文件结构

```
docs/
├── 示例文档.md          # 应用功能示例
├── 部署指南.md          # 部署说明
├── 使用说明.md          # 本文档
└── 其他文档.md          # 其他任意 Markdown 文件
```

## 预设文档特性

### 自动标签
预设文档会自动添加以下标签：
- `预设文档` - 所有预设文档的通用标签
- 基于文件路径的标签（如果在子文件夹中）
- 基于文件名的智能标签：
  - 包含"示例"或"example" → `示例`
  - 包含"指南"或"guide" → `指南`
  - 包含"教程"或"tutorial" → `教程`

### 文档属性
- `isPreset: true` - 标记为预设文档
- `originalPath` - 原始文件路径
- 自动提取的标题（从第一个 # 标题）
- 创建和更新时间

### 界面区分
- 预设文档在侧边栏单独分组显示
- 使用绿色边框和图标标识
- 不能被删除（只能编辑）
- 可以重新加载

## 使用方法

### 添加新的预设文档

1. **创建文档文件**
   ```bash
   # 在 docs 文件夹中创建新的 .md 文件
   echo "# 新文档标题" > docs/新文档.md
   ```

2. **编写内容**
   ```markdown
   # 新文档标题
   
   这里是文档内容...
   
   ## 章节标题
   
   更多内容...
   ```

3. **重新加载**
   - 开发环境：保存文件后自动更新
   - 生产环境：重新构建应用

### 管理预设文档

#### 重新加载预设文档
在侧边栏的"预设文档"区域点击刷新按钮，可以重新加载所有预设文档。

#### 编辑预设文档
- 点击预设文档旁的编辑按钮
- 在编辑器中修改内容
- 保存后更改会保存到本地存储

#### 恢复预设文档
如果预设文档被修改，可以通过重新加载功能恢复到原始状态。

### 文档组织建议

#### 文件命名
- 使用有意义的中文名称
- 避免特殊字符和空格
- 使用 `.md` 扩展名

#### 内容结构
```markdown
# 文档标题

简短的文档描述

## 主要章节

### 子章节

内容...

## 参考链接

- [相关链接](https://example.com)
```

#### 标签策略
- 利用文件名自动生成的标签
- 在文档中可以手动添加更多标签
- 保持标签的一致性和有意义

## 技术实现

### Vite 插件配置

```javascript
// vite.config.js
import { docsLoader } from './vite-plugins/docs-loader.js'

export default defineConfig({
  plugins: [
    // 其他插件...
    docsLoader(), // 文档加载插件
  ]
})
```

### 虚拟模块
```javascript
// 开发环境虚拟模块
{
  name: 'virtual-preset-docs',
  resolveId(id) {
    if (id === 'virtual:preset-docs') {
      return id
    }
  },
  load(id) {
    if (id === 'virtual:preset-docs') {
      const docs = createDevDocsLoader()
      return `export default ${JSON.stringify(docs, null, 2)}`
    }
  }
}
```

## 故障排除

### 常见问题

#### 文档没有加载
- 检查 `docs/` 文件夹是否存在
- 确认文件扩展名为 `.md`
- 查看浏览器控制台错误信息

#### 虚拟模块错误
- 重启开发服务器
- 清除浏览器缓存
- 检查 Vite 配置

#### 构建失败
- 检查 Markdown 文件语法
- 确认文件编码为 UTF-8
- 查看构建日志

### 调试方法

#### 开发环境
```javascript
// 在浏览器控制台中检查
console.log(await import('virtual:preset-docs'))
```

#### 生产环境
```javascript
// 检查生成的 JSON 文件
fetch('/preset-docs.json').then(r => r.json()).then(console.log)
```
