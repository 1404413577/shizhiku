# “重绘”和“重排”

这是一个非常棒的问题！“重绘”和“重排”是网页速度变慢的两个主要原因。

我们可以把浏览器显示网页的过程，想象成在搭一个 **乐高城堡**。

---

## 🏰 核心概念：搭乐高城堡的两个步骤

浏览器要显示一个网页，主要做两件事情：

### 1. **重排 (Reflow / Layout)**：**搭积木，量尺寸**

* **这是什么？** 浏览器计算屏幕上所有元素（文字、图片、按钮）的 **位置** 和 **大小**。
* **动作比喻：** 就像你在搭乐高，需要精确计算：
    * “这个大城堡要占多宽？”
    * “这个小旗子放在城堡的左边，离边缘 5 厘米。”
    * “如果我把左边的塔楼加高 10 厘米，右边的塔楼是不是要跟着移动？”
* **代价：** 这个步骤非常“费脑筋”。因为改变一个元素的大小或位置（比如给一个 `div` 增加宽度），**可能会影响到它周围所有的积木**，导致整个页面都需要重新计算和调整。

### 2. **重绘 (Repaint)**：**涂颜色，画图案**

* **这是什么？** 浏览器知道所有元素的位置和大小后，开始给它们 **填充颜色**、**画出边框** 或 **添加文字**。
* **动作比喻：** 就像你已经搭好了乐高城堡的结构，现在拿出画笔给它涂颜色：
    * “把屋顶涂成红色。”
    * “给窗户加上黑色的边框。”
* **代价：** 这个步骤相对“简单”一些，它只需要修改颜色，不会改变积木的位置。

---

## 🛑 什么时候会变慢？（避免重复劳动）

如果你的网页代码不断地让浏览器做这两种操作，页面就会卡住，感觉像慢动作。

### 慢的原因 1：强制重排（最耗时 🤕）

只要你修改了元素的 **大小或位置**，就会触发**重排**。

| 你的代码动作 | 浏览器被迫做的事 |
| :--- | :--- |
| 改变 `width`、`height` | **必须重排**：整个页面所有积木的位置都要重新计算。 |
| 改变 `padding`、`margin` | **必须重排**：影响了积木之间的距离，所有积木都要移动。 |
| 改变字体大小 | **必须重排**：文字变大了，可能撑开它所在的盒子。 |

> **记住：** 重排（搭积木）一定会导致重绘（涂颜色）。因为你重新搭好了，当然要重新上色。

### 慢的原因 2：只重绘（较快 🎨）

有些修改不会影响积木的位置，只会影响颜色，这只会触发 **重绘**。

| 你的代码动作 | 浏览器只做的事 |
| :--- | :--- |
| 改变 `color`（文字颜色） | **只重绘**：只换了颜料，积木位置不变。 |
| 改变 `background-color` | **只重绘**：只换了背景色，积木位置不变。 |
| 改变 `visibility: hidden` | **只重绘**：积木还在原位，只是看不见了。 |

### 🏆 结论

在写代码时，我们要尽量避免那些“大动干戈”的修改（比如改变宽度和高度），尽量使用只触发 **重绘** 的方法，或者使用现代 CSS 属性（比如 `transform`），这样你的乐高城堡才能搭得又快又流畅！
